# SKILLS.md: Project Specification for curgit

## Project Vision

`curgit` 是一个用 Rust 编写的高性能 CLI 工具，它充当独立的“Git Agent”。它分析 git 仓库中暂存的更改，并利用 LLM 生成符合 Conventional Commits 标准的专业、上下文相关的提交消息。

## Core Functional Requirements

代理必须能够执行以下操作：

1. 差异提取：通过编程方式与 Git 交互，检索暂存的更改。

2. 上下文过滤：
    - 排除干扰项（例如，锁定文件、二进制文件或 .gitignore 中指定的路径）。
    - 截断超出模型标记限制的差异，同时保留结构元数据。

3. 语义分析：
    - 识别更改的“意图”（例如，重构与错误修复）。
    - 读取根目录中的 .cursorrules 文件（如果存在），以符合本地项目编码标准。

4. 消息生成：
    - 格式：`<type>(<scope>): <subject> \n\n <body> \n\n <footer>`。
    - 支持多种语言（默认为英语，可配置为中文）。
    - 第一行首先写简单的总结，在之后使用无序列表详细介绍该 commits 的修改

5. 二进制分发：最终输出必须是静态链接的二进制文件（例如，通过 cargo build --release）。

6. 默认使用本地的 Cursor Auto 进行，并可以支持多种不同 LLM 配置（例如 Claude，GPT，Kimi 等）

7. 当提交量太大时，自动对提交进行拆分，分为多个 commits 进行描述

8. 添加一键安装的脚本服务，将其安装到一个全局的 PATH 中

9. 添加作者可以针对 LLM 生成的 commits 信息做微调或者修改功能

## 实现逻辑

AI 指令：请遵循以下架构步骤：

1. 阶段 1：Git 观察器:

- 实现一个模块，用于将索引与 HEAD 进行比较。提取：
- 已修改的文件名。
- 代码块（新增/删除的行）。
- 发生更改的函数/方法签名。

2. 阶段 2：提示工程师

- 构建一个系统提示，强制执行：
- 简洁性：无冗余信息，例如“这是提交信息”。
- 原子性：如果差异包含多个不相关的更改，建议拆分提交或警告用户。
- 风格：模仿“光标代理”的行为——智能、简洁且技术准确。

3. 阶段 3：CLI UX

- 在 API 调用期间实现加载指示器。
- 交互式确认：[Y] 提交 [E] 编辑 [N] 取消。
